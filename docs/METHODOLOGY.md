# Methodology

## Prioritisation

Four pages are scored on traffic (20%), issue count (25%), process criticality (25%), legal/HR sensitivity (15%) and feasibility (15%). Traffic and issue counts are normalised to the maximum; the other criteria are analyst judgements recorded in the published reference table.

## Cost-benefit model

Benefits comprise productivity time, assisted-support deflection, retention protection, expected legal-risk reduction and avoided future rework. Costs comprise audit, design system, development, QA, user research, change enablement, contingency and annual governance.

The three-year model discounts annual flows at 3.5%. Base NPV equals present-value gross benefits less implementation and present-value governance costs. The benefit-cost ratio is PV benefits divided by PV costs.

## Uncertainty

The published reference data preserves 1,000 triangular-input simulation draws from the source model. The Python analysis treats these as a frozen audit trail and independently reconciles the deterministic base case. Simulation outputs are decision support, not forecasts.
